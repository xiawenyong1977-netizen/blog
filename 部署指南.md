# GitHub Pages博客部署指南

## 🚀 快速开始（5分钟）

### 步骤1：创建GitHub仓库

1. 登录GitHub
2. 创建新仓库，命名为：`xintuxiangce-blog`
3. 设置为Public（GitHub Pages免费版需要Public仓库）
4. 不要初始化README、.gitignore或license

### 步骤2：上传博客文件

```bash
# 进入blog目录
cd blog

# 初始化Git仓库
git init
git add .
git commit -m "Initial commit: GitHub Pages blog"

# 添加远程仓库（替换YOUR_USERNAME为您的GitHub用户名）
git remote add origin https://github.com/YOUR_USERNAME/xintuxiangce-blog.git

# 推送到GitHub
git branch -M main
git push -u origin main
```

### 步骤3：启用GitHub Pages

1. 进入GitHub仓库页面
2. 点击 **Settings** → **Pages**
3. 在 **Source** 部分：
   - 选择 **Deploy from a branch**
   - Branch选择 **main**
   - Folder选择 **/ (root)**
4. 点击 **Save**

### 步骤4：访问博客

- **默认地址**：`https://YOUR_USERNAME.github.io/xintuxiangce-blog`
- **或配置自定义域名**：`https://blog.xintuxiangce.top`（见下方配置）

## 🌐 配置自定义域名

### 步骤1：配置DNS

在您的域名DNS管理中添加CNAME记录：

```
类型: CNAME
名称: blog
值: YOUR_USERNAME.github.io
TTL: 3600
```

### 步骤2：在GitHub中配置

1. 在仓库根目录已经有 `CNAME` 文件，内容为：`blog.xintuxiangce.top`
2. 如果还没有，创建 `CNAME` 文件：
   ```bash
   echo "blog.xintuxiangce.top" > CNAME
   git add CNAME
   git commit -m "Add CNAME for custom domain"
   git push
   ```

3. 在GitHub仓库设置中：
   - Settings → Pages → Custom domain
   - 输入：`blog.xintuxiangce.top`
   - 勾选 **Enforce HTTPS**

### 步骤3：等待DNS生效

DNS记录通常需要几分钟到几小时生效。可以使用以下命令检查：

```bash
# 检查DNS记录
nslookup blog.xintuxiangce.top

# 或使用在线工具
# https://dnschecker.org/
```

## 📝 添加新文章

### 创建文章文件

在 `_posts` 目录中创建新文件，文件名格式：`YYYY-MM-DD-文章标题.md`

```bash
# 示例
_posts/2025-12-16-AI智能分类技术详解.md
```

### 文章模板

```markdown
---
layout: post
title: "文章标题"
date: 2025-12-16
categories: [分类1, 分类2]
tags: [标签1, 标签2]
author: 芯图团队
---

## 文章内容

这里是文章正文...

## 添加指向主网站的链接

了解更多，请访问[芯图相册官网](https://www.xintuxiangce.top) - AI智能照片分类工具。

<!-- 这个链接是dofollow，会传递SEO权重！ -->
```

### 发布文章

```bash
# 添加文件
git add _posts/2025-12-16-AI智能分类技术详解.md

# 提交
git commit -m "Add: AI智能分类技术详解"

# 推送
git push
```

GitHub Actions会自动构建和部署，通常几分钟内就能看到更新。

## 🔗 外链策略

### 在文章中添加dofollow链接

所有在博客文章中的链接默认都是**dofollow**，会传递SEO权重：

```markdown
<!-- 好的链接方式 -->
[芯图相册](https://www.xintuxiangce.top)是一款AI智能照片分类工具。

<!-- 更好的方式：使用关键词作为锚文本 -->
[照片分类工具](https://www.xintuxiangce.top) - AI智能分类，90%+准确率。

<!-- 避免的方式 -->
点击[这里](https://www.xintuxiangce.top)了解更多。
```

### 链接位置建议

- ✅ **正文中**：权重最高
- ✅ **相关文章推荐**：权重较高
- ⚠️ **侧边栏**：权重较低
- ❌ **页脚**：权重最低

### 每篇文章的链接数量

- ✅ **2-3个链接**：自然、不过度
- ❌ **避免**：10+个链接（会被视为垃圾链接）

## 🔍 SEO优化

### 1. 文章标题

- ✅ 包含关键词：如"照片分类"、"AI智能分类"
- ✅ 长度控制在50-60字符
- ✅ 吸引点击：使用数字、疑问句等

### 2. 文章描述

在Front Matter中添加description：

```yaml
---
title: "照片分类完全指南"
description: "详细介绍8种照片分类方法，帮助您高效整理照片。芯图相册支持AI智能分类，90%+准确率。"
---
```

### 3. 关键词使用

- ✅ 主关键词在标题中出现1次
- ✅ 主关键词在正文中自然出现8-12次
- ✅ 使用相关关键词和同义词

### 4. 内部链接

- ✅ 链接到主网站的相关页面
- ✅ 链接到博客中的其他相关文章
- ✅ 使用关键词作为锚文本

## 🆘 常见问题

### Q: 博客无法访问？

A: 检查：
1. GitHub Pages是否已启用
2. 仓库是否为Public
3. 构建是否成功（查看Actions标签）

### Q: 自定义域名不工作？

A: 检查：
1. DNS记录是否正确配置
2. CNAME文件是否存在
3. GitHub Pages设置中是否配置了自定义域名
4. 等待DNS生效（可能需要几小时）

### Q: 文章不显示？

A: 检查：
1. 文件是否在 `_posts` 目录
2. 文件名格式是否正确：`YYYY-MM-DD-标题.md`
3. Front Matter格式是否正确
4. 构建是否成功

### Q: 链接不传递权重？

A: 确认：
1. 链接没有 `rel="nofollow"` 属性
2. 链接在正文中（不是页脚）
3. 链接周围有相关上下文
